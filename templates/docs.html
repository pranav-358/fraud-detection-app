üìã Overview

FraudShield is an AI-powered fraud detection system that analyzes transaction patterns in real-time to identify potentially fraudulent activity. Built with Python, Flask, and Scikit-Learn, it provides instant fraud risk assessment with 99.2% accuracy.

Demo Purpose: FraudShield is designed for educational and demonstration purposes. It showcases machine learning capabilities but should not be used as the sole fraud detection system for real financial transactions.

üîç How It Works

FraudShield uses a Random Forest machine learning algorithm to analyze three key transaction features:

Input Features

  1. Transaction Amount: The monetary value of the transaction in USD
  2. Distance from Home: How far (in kilometers) the transaction occurs from the user's home location
  3. Transaction Type: The method of transaction (Online, In-Store, or ATM)

Analysis Process

  1. Data Input: User enters transaction details in the web interface
  2. Preprocessing: Data is normalized using StandardScaler
  3. ML Prediction: Random Forest model analyzes patterns
  4. Risk Assessment: Model returns fraud probability and confidence level
  5. Result Display: User receives instant feedback with visual indicators

ü§ñ Machine Learning Model

Algorithm: Random Forest Classifier

Random Forest is an ensemble learning method that operates by constructing multiple decision trees and outputting the mode of their predictions.

Model Specifications

Parameter Value
Algorithm Random Forest
Number of Estimators 100
Max Depth 10
Training Samples 4,000
Test Samples 1,000
Training Accuracy ~99.5%
Test Accuracy ~99.2%

Training Data

The model is trained on 5,000 synthetic transactions:

  • Legitimate Transactions (80%): Average amount ~$75, close to home (<50km), varied types
  • Fraudulent Transactions (20%): Higher amounts (~$300), far from home (50-500km), more online purchases

Note: The model is trained on synthetic data for demonstration purposes. Real-world fraud detection systems require much larger datasets with real transaction patterns.

üìñ Usage Guide

Using the Web Interface

  1. Navigate to FraudShield: Open the website in your browser
  2. Enter Transaction Amount: Input the dollar amount (e.g., 150.00)
  3. Enter Distance: Input distance from home in kilometers (e.g., 25)
  4. Select Transaction Type: Choose Online, In-Store, or ATM
  5. Click "Analyze Transaction": Submit for analysis
  6. View Results: See fraud probability, confidence level, and recommendation

Interpreting Results

  • Green/Safe: Transaction appears legitimate (low fraud probability)
  • Red/Warning: High fraud risk detected (high fraud probability)
  • Confidence Level: How certain the model is about its prediction (0-100%)
  • Fraud Probability: Percentage chance the transaction is fraudulent
